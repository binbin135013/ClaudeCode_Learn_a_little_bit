  Claude Code (Claw Code) 核心能力与"环境设计"深度解析

  一、项目概览

  本项目是 Claude Code 的 Rust 重写版（Claw Code），采用 workspace 模式组织为 5
  个 crate：

  rust/crates/
  ├── api/          # Anthropic API 客户端（流式/非流式）
  ├── runtime/      # 运行时核心（权限、沙箱、会话、Hook、上下文压缩）
  ├── tools/        # 20+ 工具定义与执行引擎
  ├── commands/     # 命令注册
  └── rusty-claude-cli/  # CLI 入口与 TUI 渲染

  三、"环境设计"——让 AI 更可靠的 6 层防护

  Claude Code 的核心设计哲学不是让 AI 变得更聪明，而是通过环境约束让 AI 
  的行为更可预测、更安全。

  第 1 层：分级权限系统 (permissions.rs)

  ReadOnly → WorkspaceWrite → DangerFullAccess
               ↑
             Prompt（交互式提升）

  每个工具声明所需权限级别，运行时强制执行：

  // tools/src/lib.rs:77-78 — bash 需要 DangerFullAccess
  ToolSpec { name: "bash", required_permission:
  PermissionMode::DangerFullAccess, ... }
  // tools/src/lib.rs:89-90 — read_file 只需 ReadOnly
  ToolSpec { name: "read_file", required_permission: PermissionMode::ReadOnly,
  ... }

  当 AI 尝试使用高权限工具时，系统会弹出用户确认（PermissionPrompter
  trait），用户可逐个允许或拒绝。

  第 2 层：Hook 系统 (hooks.rs)

  在工具执行前后插入可编程的安全检查：

  // hooks.rs — Hook 执行流程
  PreToolUse hook → exit 0 (允许) / exit 2 (拒绝) / 其他 (警告但允许)
    → 执行工具
  PostToolUse hook → 注入反馈到工具结果中

  这意味着团队可以配置规则如"禁止编辑 .env 文件"、"每次 bash
  执行前记录日志"等，AI 无法绕过。

  第 3 层：上下文感知提示词 (prompt.rs)

  系统提示词不是静态的，而是动态构建的：

  // prompt.rs:134-156 — SystemPromptBuilder
  sections.push(get_simple_intro_section());      // 角色定义
  sections.push(get_simple_system_section());      // 行为约束
  sections.push(get_simple_doing_tasks_section()); // 任务执行规范
  sections.push(self.environment_section());       // 环境信息（OS、路径、日期）
  sections.push(render_project_context(...));      // Git 状态、diff 快照
  sections.push(render_instruction_files(...));    // CLAUDE.md 层级发现

  关键设计：
  - CLAUDE.md 层级发现（行 192-213）：从项目根到当前目录，逐层加载
  CLAUDE.md、CLAUDE.local.md、.claude/instructions.md
  - Git 上下文注入（行 227-275）：自动运行 git status 和 git diff，让 AI
  知道当前代码变更状态
  - 内容去重与截断（行 326-380）：相同内容的 CLAUDE.md
  只加载一次，超大文件自动截断

  第 4 层：自动上下文压缩 (compact.rs)

  长对话会超出上下文窗口。系统自动压缩旧消息：

  // compact.rs:75-111
  pub fn compact_session(session: &Session, config: CompactionConfig) ->
  CompactionResult {
      // 保留最近 N 条消息，将旧消息压缩为摘要
      let removed = &session.messages[..keep_from];
      let preserved = session.messages[keep_from..].to_vec();
      let summary = summarize_messages(removed);
      // 摘要包含：消息统计、使用的工具、待办工作、关键文件、时间线
  }

  压缩后的摘要包含：消息统计、使用的工具列表、最近用户请求、待办工作推断、关键文
  件引用、完整时间线。

  第 5 层：工具执行沙箱

  - bash 工具支持 timeout 参数和 run_in_background 模式
  - edit_file 强制 old_string != new_string（防止空操作）
  - WebFetch 自动将 HTTP 升级为 HTTPS（防止中间人攻击）
  - URL 验证：只允许 localhost/127.0.0.1 使用 HTTP

  第 6 层：子 Agent 隔离

  每个子 Agent 在独立线程中运行，拥有：
  - 独立的 Session（无法访问主会话消息）
  - 受限的工具集（Explore 类型甚至不能执行 bash）
  - 最大迭代次数限制（DEFAULT_AGENT_MAX_ITERATIONS = 32）
  - panic 捕获和优雅的错误恢复

  总结：环境设计的核心原则

  ┌───────────┬───────────────────────────────────┬─────────────────────────┐
  │   原则    │             实现方式              │          效果           │
  ├───────────┼───────────────────────────────────┼─────────────────────────┤
  │ 最小权限  │ 每个工具声明权限级别，运行时强制  │ AI 只能做被允许的事     │
  │           │ 执行                              │                         │
  ├───────────┼───────────────────────────────────┼─────────────────────────┤
  │ 可审计性  │ Hook 系统在工具执行前后插入检查点 │ 每个操作都有记录和拦截  │
  │           │                                   │ 机会                    │
  ├───────────┼───────────────────────────────────┼─────────────────────────┤
  │ 上下文感  │ 自动加载 CLAUDE.md、Git           │ AI 始终了解项目约定     │
  │ 知        │ 状态、项目配置                    │                         │
  ├───────────┼───────────────────────────────────┼─────────────────────────┤
  │ 故障隔离  │ 子 Agent                          │ Agent 失败不影响主会话  │
  │           │ 独立会话、受限工具、最大迭代限制  │                         │
  ├───────────┼───────────────────────────────────┼─────────────────────────┤
  │ 优雅降级  │ 上下文压缩保留摘要而非丢失        │ 长对话不崩溃            │
  ├───────────┼───────────────────────────────────┼─────────────────────────┤
  │ 人机协作  │ Prompt 模式让用户审批高风险操作   │ 关键决策始终由人做      │
  └───────────┴───────────────────────────────────┴─────────────────────────┘

  这套设计让 AI 不需要"更聪明"，而是在一个结构良好、约束清晰的环境中工作，从而大
  幅提高编程任务的可靠性和安全性。

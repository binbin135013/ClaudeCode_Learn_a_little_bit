# 记忆系统 (Memory)

> [!summary] 一句话定义
> AI 跨会话记住信息的能力。Claude Code 和 OpenClaw 各有实现，但都基于**本地文件存储**。

---

## OpenClaw 记忆系统

> [!note] 设计哲学：简单到极致
> 没有向量数据库、没有 RAG 管道，就是 Markdown 文件。

### 两层架构

| 层级 | 文件 | 内容 | 加载时机 |
|------|------|------|----------|
| **长期记忆** | `MEMORY.md` | 偏好、决策、关键事实 | 每次会话启动 |
| **每日日志** | `memory/YYYY-MM-DD.md` | 当天笔记和上下文 | 今天 + 昨天 |

```
~/.openclaw/workspace/
├── MEMORY.md              # 长期记忆（精选重要信息）
└── memory/
    ├── 2026-04-02.md      # 今天的日志
    └── 2026-04-01.md      # 昨天的日志
```

### 记忆工具

| 工具 | 功能 |
|------|------|
| `memory_search` | 语义搜索所有记忆文件 |
| `memory_get` | 精确读取特定记忆内容 |

### 优势

- **可读** — 随时打开 Markdown 查看 AI 记住了什么
- **可编辑** — 直接修改记忆文件
- **可版本控制** — 放进 Git 追踪变化
- **零依赖** — 不需要额外数据库

## Claude Code 记忆

### 原生机制

- **CLAUDE.md** — 项目级持久化指令 → [[CLAUDE-md编写指南]]
- **Auto Memories** — 自动记忆功能（v2.1.78+）
- **`.claude/` 目录** — 配置和状态持久化

### oh-my-claudecode 增强

- **Priority Context** — 永久加载的优先上下文
- **Working Memory** — 7 天自动清理的工作记忆
- **Project Memory** — 跨会话持久化的项目知识

## 进阶：记忆优化策略

### OpenClaw 记忆分层策略

> [!tip] 记忆分层原则
> 不是所有信息都值得记住。按重要性分层：

| 层级 | 内容 | 存储位置 | 示例 |
|------|------|---------|------|
| **黄金** | 核心偏好和决策 | MEMORY.md | "我习惯用 TypeScript strict 模式" |
| **白银** | 近期有用信息 | 当日日志 | "今天在调试 auth 模块的 JWT 问题" |
| **青铜** | 临时上下文 | 不存储 | "帮我查一下现在几点了" |

### MEMORY.md 最佳实践

```markdown
# 我的记忆

## 个人偏好
- 语言：中文交流，英文代码
- 编程风格：函数式优先，避免 class
- 沟通风格：简洁直接，不要废话

## 重要决策
- 2026-03-15：项目 X 决定用 Vite 而非 Webpack
- 2026-04-01：团队统一用 pnpm 代替 npm

## 常用联系人/项目
- 项目 A：GitHub repo URL
- 项目 B：部署地址
```

### Claude Code 记忆增强

**Auto Memories (v2.1.78+)**：
Claude Code 现在会自动将重要信息写入记忆，无需手动配置。

**oh-my-claudecode 记忆体系**：
- **Priority Context** — 始终加载的优先信息（< 500 字符）
- **Working Memory** — 7 天自动清理的工作记忆
- **Project Memory** — 跨会话持久化的项目知识

> [!note] CC vs OC 记忆对比
> | 维度 | Claude Code | OpenClaw |
> |------|------------|----------|
> | 存储 | ~/.claude/ + CLAUDE.md | Markdown 文件 |
> | 加载 | 每次会话 | 每次会话 |
> | 搜索 | 内置 Grep | memory_search 语义搜索 |
> | 可读性 | JSON + MD | 纯 Markdown |
> | 人工编辑 | 可编辑 | 可编辑 |

# 消息平台集成

> [!summary] 一句话定义
> OpenClaw 通过 **Channel 抽象层** 统一接入 15+ 消息平台，一个 AI 助手通过所有常用聊天软件都能访问。

---

## 核心平台详细配置

### Telegram（推荐入门）

> [!example] 配置步骤
> 1. 与 @BotFather 对话，创建 Bot 获取 Token
> 2. 在 `openclaw.json` 中配置：
> ```json5
> {
>   "channels": {
>     "telegram": {
>       "botToken": "123456:ABC-DEF..."
>     }
>   }
> }
> ```
> 3. 重启 Gateway：`openclaw gateway --verbose`
> 4. 向 Bot 发送消息测试

**Telegram 优势**：
- 配置最简单（只需一个 Bot Token）
- 群聊支持最好
- 支持内联键盘、富文本
- 支持 Bot Commands 菜单

### WhatsApp

> [!warning] WhatsApp 注意事项
> - 使用 Baileys Web 协议（非官方 API）
> - 首次连接需扫码配对
> - 号码可能被封（风险约 1-2%）
> - 建议用备用号码

### Discord

配置要点：
1. 创建 Discord Application → Bot
2. 开启 Message Content Intent（必需！）
3. 邀请 Bot 到服务器（OAuth2 URL Generator）
4. 配置 Bot Token

### WebChat（调试利器）

> [!tip] 调试技巧
> WebChat 是内置的浏览器聊天界面，无需额外配置：
> - 访问 `http://localhost:18789` 即可使用
> - 支持 Markdown 渲染
> - 适合开发调试时快速测试
> - 不需要外部平台账号

## DM 配对机制

```
陌生人发消息 → OpenClaw 返回配对码
                     ↓
终端确认：openclaw pairing approve <platform> <code>
                     ↓
该用户被加入允许列表，后续消息正常处理
```

> - 同一个 [[技能系统]] — 同一套技能可用

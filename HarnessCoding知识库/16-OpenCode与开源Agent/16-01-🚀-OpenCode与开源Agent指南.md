## 什么是 OpenCode/Code Agent

### 定义

OpenCode（开放代码代理）是指具备自主执行代码任务能力的 AI Agent。与传统 CLI 工具不同，Code Agent 通常具备：

- **多步骤推理**: 能够规划复杂任务的执行路径
- **工具调用**: 主动调用文件系统、终端、浏览器等工具
- **上下文理解**: 理解项目结构、代码库语义
- **自我修正**: 根据执行结果反馈调整策略

### Claude Code vs 其他 Code Agent

> [!note] 定位对比
> | 维度 | Claude Code | OpenCode 项目 |
> |------|------------|---------------|
> | **核心优势** | 深度推理、安全可靠 | 各有特色（速度/生态/语言） |
> | **交互方式** | 终端 CLI | 多样化（API/聊天/IDE插件） |
> | **开源程度** | 闭源（Anthropic） | 源码开放、可自托管 |
> | **定制灵活性** | Hook/配置有限 | 完全可修改 |
> | **生态丰富度** | MCP 生态 | 各有扩展生态 |

### 常见 Code Agent 类型

1. **通用代码助手**: 如 Claude Code、Copilot Workspace
2. **自动化执行引擎**: 如 OpenClaw、AutoGPT
3. **代码审查专精**: 如 CodeRabbit、SWE-agent
4. **全栈开发代理**: 如 Devin、Cognition

## 为什么学习 OpenCode 项目

### 1. 深入理解 Agent 架构

通过阅读不同实现，理解：
- Tool 系统设计模式
- 任务分解与规划策略
- 上下文管理方案
- 错误处理与恢复机制

### 2. 获取灵感与最佳实践

各开源项目往往在特定领域有创新：
- 提示词工程
- 工具调用编排
- 记忆与状态管理
- 多 Agent 协作

### 3. 定制化需求

开源项目可以满足：
- 私有化部署需求
- 特定工作流集成
- 品牌定制和 UI 定制
- 成本优化（使用自有模型）

### 4. 跟踪技术前沿

Code Agent 领域发展迅速：
- 新技术路线探索
- 新工具调用范式
- 新评估标准建立

## 学习路径建议

### 阶段一：入门概览（1-2天）

```
1. 了解 Code Agent 基本概念
2. 体验 2-3 个主流开源项目
3. 对比与 Claude Code 的差异
```

**推荐项目**: OpenClaw、SWE-agent

### 阶段二：深入源码（2-3天）

```
1. 选择一个感兴趣的项目
2. 阅读核心模块源码
3. 理解架构设计和决策
```

**推荐项目**: [[16-03-🐍-claw-code-Python重写]]、[[16-04-🦀-claw-code-parity-Rust实现]]、[[16-05-⚡-omx编排工作流]]

### 阶段三：实践应用（持续）

```
1. 尝试定制和扩展
2. 解决实际问题
3. 贡献和改进项目
```

# ♻️ 核心设计模式

## 1. 工厂模式 - buildTool

```mermaid
flowchart TB
    subgraph Client
        C[调用方]
    end

    subgraph Factory
        F[buildTool]
        FC[防御性检查]
        FF[createFallbackTool]
    end

    subgraph Products
        P1[正常Tool]
        P2[Fallback Tool]
    end

    C --> F
    F --> FC
    FC -->|成功| P1
    FC -->|异常| FF
    FF --> P2
```

**要点**：工厂封装了创建细节，调用方无需关心实现。

## 3. 拦截器模式 - Hook

```mermaid
sequenceDiagram
    participant C as Caller
    participant H as withHook
    pre as Pre-Hook
    fn as Core Function
    post as Post-Hook

    C->>H: 调用函数
    H->>pre: emit(beforeEvent)

    alt 有拦截器阻止
        pre-->>H: {blocked: true}
        H-->>C: 返回 blocked
    else 继续执行
        pre-->>H: 允许
        H->>fn: 执行核心逻辑
        fn-->>H: result
        H->>post: emit(afterEvent, result)
        post-->>H: 确认
        H-->>C: 返回 result
    end
```

**要点**：在不修改核心逻辑的情况下增强行为。

## 5. 断路器模式 - AutoCompact

```mermaid
stateDiagram-v2
    [*] --> Closed
    Closed --> Open : 失败次数 >= threshold
    Open --> HalfOpen : 超时后尝试
    HalfOpen --> Closed : 成功
    HalfOpen --> Open : 失败

    note right of Closed: 正常执行<br/>每次成功-1失败
    note right of Open: 快速失败<br/>拒绝请求
    note right of HalfOpen: 探针测试<br/>允许部分请求
```

```typescript
class CircuitBreaker {
  private failures = 0;
  private readonly threshold = 3;

  async execute(fn: () => Promise<Result>) {
    if (this.failures >= this.threshold) {
      throw new CircuitOpenError();
    }
    try {
      return await fn();
    } catch (e) {
      this.failures++;
      throw e;
    }
  }
}
```

**要点**：防止级联失败，快速失败而非无限重试。

## 6. 消息累积模式 - QueryEngine

```mermaid
flowchart TB
    subgraph QueryEngine
        MM[mutableMessages]
        S[send]
        AI[callAI]
    end

    M1(Message 1) --> S
    M2(Message 2) --> S
    M3(Message 3) --> S

    S -->|push| MM
    MM -->|发送| AI
    AI -->|追加response| MM

    subgraph MessageHistory
        M1
        M2
        M3
        R1(Response 1)
        R2(Response 2)
    end

    MM --> R1
    R1 --> R2
```

```typescript
class QueryEngine {
  private mutableMessages: Message[] = [];

  async send(message: Message) {
    this.mutableMessages.push(message);
    const response = await this.callAI(this.mutableMessages);
    this.mutableMessages.push(response);
    return response;
  }
}
```

**要点**：跨调用保持状态。

## 7. 四层缓存 - 上下文压缩

```mermaid
flowchart LR
    subgraph Levels
        L1[Level 1: Snip<br/>清除内容标记]
        L2[Level 2: Micro<br/>缓存失效触发]
        L3[Level 3: Collapse<br/>选择性重构]
        L4[Level 4: Auto<br/>LLM生成摘要]
    end

    Cost0[$0] --> Cost1[<$0.01]
    Cost1 --> Cost2[<$0.10]
    Cost2 --> Cost3[<$0.50]

    L1 --> L2
    L2 --> L3
    L3 --> L4

    style L1 fill:#90EE90
    style L2 fill:#FFFF99
    style L3 fill:#FFA500
    style L4 fill:#FF6B6B
```

| 级别 | 策略 | 成本 |
-----|------|-----|
| L1 Snip | 清除内容标记 | 0 |
| L2 Micro | 缓存失效触发 | 极低 |
| L3 Collapse | 选择性重构 | 中 |
| L4 Auto | LLM生成摘要 | 高 |

**要点**：渐进式优化，而非一步到位。

## 8. Fork/Join 模式 - 子Agent

```mermaid
flowchart TB
    subgraph Parent
        P[主Agent]
        Pm[任务分发]
        Pr[结果收集]
    end

    subgraph Workers
        A[Agent A]
        B[Agent B]
        C[Agent C]
    end

    subgraph Execution
        E1[并行执行]
        E2[等待完成]
        E3[结果合并]
    end

    P --> Pm
    Pm -->|fork| A
    Pm -->|fork| B
    Pm -->|fork| C

    A --> E1
    B --> E1
    C --> E1

    E1 --> E2
    E2 -->|join| E3
    E3 --> Pr
    Pr --> P
```

**要点**：并行处理任务，最后合并结果。

## 10. 模式关系图

```mermaid
flowchart TB
    subgraph Core
        E[事件驱动]
        F[工厂模式]
    end

    subgraph Extension
        I[拦截器]
        Pi[管道]
    end

    subgraph Reliability
        C[断路器]
        Ca[四层缓存]
    end

    subgraph State
        M[消息累积]
        FJ[Fork/Join]
    end

    E --> I
    E --> Pi
    F --> I
    F --> M
    I --> C
    Pi --> Ca
    M --> FJ
    Ca --> FJ

    style E fill:#4169E1,color:#fff
    style F fill:#4169E1,color:#fff
    style I fill:#90EE90
    style Pi fill:#90EE90
    style C fill:#FFA500
    style Ca fill:#FFA500
    style M fill:#FF6B6B
    style FJ fill:#FF6B6B
```

**模式协作关系**：
- **事件驱动**是基础设施，拦截器和管道基于它构建
- **工厂**创建拦截器和管道组件
- **断路器**保护拦截器调用
- **消息累积**依赖工厂创建的消息队列
- **四层缓存**为消息累积提供优化
- **Fork/Join**组合消息累积的结果

# 🚀 Claude Code 新手入门完全指南

> ⏱ 预计阅读时间: 15分钟 | 🎯 目标: 让你今天就能用起来

## 你能学到什么

- ✅ 启动 Claude Code 并完成第一次对话
- ✅ 理解 `/help` 和 `/commands` 的用法
- ✅ 学会让 AI 帮你读写文件
- ✅ 理解权限提示是什么
- ✅ 创建你的第一个项目

---

## 第一步：启动 Claude Code

### 启动命令

```bash
# 在你的项目目录下启动
cd ~/your-project
claude
```

### 首次启动会发生什么

1. **权限初始化** - 你会看到权限说明
2. **模型选择** - 可选使用特定模型
3. **欢迎信息** - 显示基本命令

### 退出对话

```bash
/exit    # 退出
/clear   # 清除会话，重新开始
```

---

## 第二步：理解基本命令

Claude Code 使用 `/` 开头的命令。

### 必须知道的命令

| 命令 | 作用 | 示例 |
|-----|------|-----|
| `/help` | 显示帮助 | `/help` |
| `/exit` | 退出 | `/exit` |
| `/clear` | 清除会话 | `/clear` |
| `/commit` | 创建 git 提交 | `/commit` |
| `/plan` | 显示执行计划 | `/plan` |
| `/review` | 代码审查 | `/review` |

### 如何找到更多命令

```bash
/help commands    # 列出所有命令
/help permissions # 查看权限相关
```

## 第三步：让 AI 帮你工作

### 读文件

```
请帮我读取 src/index.js 文件
```

Claude Code 会使用 [[../02-Tool系统/02-01-🔧-Tool系统]] 中的 Read 工具读取文件。

### 写文件

```
请帮我创建一个 index.html 文件，内容是一个简单的页面
```

### 编辑文件

```
请帮我修改 src/config.js，把端口号改成 3000
```

### 执行命令

```
请帮我运行 npm install
```

## 第四步：理解权限提示 🔐

当你执行敏感操作时，Claude Code 会询问权限。

### 常见权限提示

**文件写入**
```
⚠️ 权限请求: Write
文件: src/new-file.js
是否允许? [y/n]
```

**Bash 执行**
```
⚠️ 权限请求: Bash
命令: rm -rf node_modules
是否允许? [y/n]
```

### 如何响应

| 输入 | 行为 |
|-----|------|
| `y` | 允许这一次 |
| `n` | 拒绝 |
| `a` | 全部允许（危险!） |
| `p` | 永久允许 |

### 信任级别设置

```bash
# 查看当前信任级别
/settings trust

# 设置信任的工作区
/trust ~/my-project
```

## 第五步：你的第一个项目

### 场景：创建一个笔记应用

**1. 启动 Claude Code**
```bash
cd ~/projects
claude
```

**2. 开始对话**
```
我想创建一个简单的笔记应用，使用 Node.js 和 SQLite。请帮我：
1. 设计项目结构
2. 创建数据库模块
3. 创建 API 路由
4. 创建前端页面
```

**3. Claude Code 会：**
1. 📐 制定计划 - 展示将要做的步骤
2. 🔧 执行工具 - 读写文件、执行命令
3. 🔐 请求权限 - 需要时询问
4. ✅ 完成交付 - 展示结果

## 常见问题 FAQ

### Q: Claude Code 和 ChatGPT 有什么区别？

| | Claude Code | ChatGPT |
|--|------------|---------|
| 运行环境 | 本地文件系统 | 云端 |
| 工具调用 | 可以读写文件、执行命令 | 不能 |
| 权限控制 | 有权限系统 | 无 |
| 适合场景 | 开发工作 | 对话问答 |

### Q: 权限提示太多了怎么办？

**选项1：设置信任的工作区**
```
/trust ~/my-trusted-project
```

**选项2：使用 plan 模式（只读）**
```
/plan
```
这个模式下只读取文件，不执行写入操作。

**选项3：使用 --dangerously-skip-permissions（危险!）**
```bash
claude --dangerously-skip-permissions
```
⚠️ 仅在完全信任的环境下使用！

### Q: 我的文件会被永久保存吗？

是的！Claude Code 可以：
- 📝 创建和修改文件
- 💾 提交到 git
- 📦 安装 npm 包

所有操作都会持久化到你的文件系统中。

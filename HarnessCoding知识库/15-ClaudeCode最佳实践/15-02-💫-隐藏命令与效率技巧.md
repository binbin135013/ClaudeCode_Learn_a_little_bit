# Claude Code 隐藏命令与效率技巧

> ⏱ 适用水平: 入门到高级 | 推荐程度: ⭐⭐⭐ 必读
> 掌握这些隐藏命令和技巧，可显著提升开发效率和工作流灵活性

---

## 1. 对话与工作流管理

### 1.1 /rewind - 智能回退

```bash
/rewind
```

**功能**: 回退到对话的早期状态，重新开始当前任务。Claude Code 会保留最近的对话历史以便对比。

**使用场景**:
- 任务方向走偏，需要重新开始
- 想尝试不同的实现方式
- 对话上下文变得过于复杂

> [!NOTE]
> `/rewind` 不支持 `--code`、`--chat`、`--all` 等参数标志。

---

### 1.2 /branch - 创建会话分支

```bash
/branch [分支名称]
```

**功能**: 从当前对话点创建平行会话分支，探索不同方案而不影响原会话。

**使用场景**:
- 想尝试两种不同的实现方案
- 需要探索性开发但不想污染主线
- A/B 测试不同的代码风格

**工作流**:
```
主会话: 正在开发功能 A
  → /branch feature-a-optimization
  → 在分支中尝试优化方案
  → 满意则合并回主会话
  → 不满意则直接关闭分支
```

---

### 1.3 /export - 导出对话

```bash
/export [文件名]
```

**功能**: 将对话导出为结构化 Markdown 文件，用于存档或交给其他 AI 工具分析。

**使用场景**:
- 项目复盘时保存决策过程
- 与团队分享讨论内容
- 将上下文交给其他 AI 工具继续处理

## 2. OMC/插件命令 (非原生 Claude Code)

以下命令来自 OMC (oh-my-claudecode) 框架或 Claude Code 插件，不是原生 Claude Code 命令。

### 2.1 /btw - 插入并行问题 (OMC)

```bash
/btw [你的问题]
```

**功能**: 在执行耗时任务时插入不相关问题，独立处理，不污染主任务历史。

**使用场景**:
- 主任务正在运行代码审查时，突然想到一个配置问题
- 长任务执行中需要临时查询某些内容
- 不希望某个问题打断当前工作流

**示例**:
```
# 主任务: 重构用户模块
正在重构用户认证系统...
# 此时使用:
/btw 如何配置 Git hooks 自动格式化代码？
# Claude 会独立处理这个问题，完成后返回主任务
```

---

### 2.2 /simplify - 代码审查 (OMC Skill)

```bash
/simplify [文件路径或代码段]
```

**功能**: 三合一代码审查，自动分析：
- **代码复用**: 重复代码检测与重构建议
- **代码质量**: 潜在 bug、性能问题、安全隐患
- **运行效率**: 性能优化建议

**输出**: 详细的优化建议报告

**使用场景**:
- 代码审查时快速定位问题
- 重构前评估代码健康度
- 优化前确定优先级

> [!TIP]
> 此命令是 OMC 框架的 Skill，不是原生 Claude Code 命令。

---

### 2.3 /loop - 定时循环执行 (OMC Skill)

```bash
/loop [间隔时间] [任务描述]
```

**功能**: 定时循环执行任务，适合监控部署状态或定期生成报告。

**参数**:
- `间隔时间`: 如 `5m`（5分钟）、`1h`（1小时）、`1d`（1天）

**使用场景**:
- 监控部署状态直到完成
- 定期检查服务器健康状态
- 定时生成项目报告

**示例**:
```
/loop 10m 检查部署是否完成
# 每 10 分钟执行一次，直到部署成功或手动停止
```

> [!TIP]
> 此命令是 OMC 框架的 Skill，不是原生 Claude Code 命令。

---

### 2.4 /schedule - 管理计划任务 (OMC Skill)

```bash
/schedule [时间规则] [任务]
```

**功能**: 管理长期计划任务，支持复杂时间规则。

**时间规则示例**:
| 规则 | 说明 |
|-----|------|
| `daily 09:00` | 每天上午 9 点 |
| `weekly monday` | 每周一 |
| `0 9 * * *` | Cron 表达式 |

**使用场景**:
- 每日 standup 报告生成
- 每周代码统计汇总
- 定时健康检查

> [!TIP]
> 此命令是 OMC 框架的 Skill，不是原生 Claude Code 命令。

---

### 2.5 /insights - 使用分析报告 (OMC Skill)

```bash
/insights
```

**功能**: 生成个性化报告，分析：
- 使用习惯
- 命令频率
- 重复操作模式
- 优化建议

**使用场景**:
- 了解自己的使用模式
- 发现效率瓶颈
- 获取个性化改进建议

> [!TIP]
> 此命令是 OMC 框架的 Skill，不是原生 Claude Code 命令。

---

### 2.6 /xlsx / docx - 文件处理 (Skill)

这些是 Claude Code 的文档处理 Skill，需要通过 `/xlsx` 或 `/docx` 激活。

```bash
/xlsx [文件路径]    # 处理 Excel 文件
/docx [文件路径]    # 处理 Word 文件
```

**功能**: 直接处理 Office 文件。

**使用场景**:
- 分析 Excel 数据生成报告
- 编辑 Word 文档

> [!TIP]
> 这些是 Skill 插件命令，不是原生 Claude Code 命令。

## 3. 快捷键与效率提升

### 3.1 核心快捷键

| 快捷键 | 功能 | 重要性 |
|-------|------|--------|
| `Ctrl + V` | 直接粘贴图片，Claude 可识别截图中的代码或错误信息 | ⭐⭐⭐ |
| `Ctrl + J` | 多行输入换行 | ⭐⭐⭐ |
| `Option + Enter` | 多行输入换行（macOS） | ⭐⭐⭐ |

### 3.2 高级操作

| 操作 | 说明 |
|-----|------|
| `Ctrl + C` (连按) | 停止当前任务 |
| `Ctrl + D` | 结束当前会话 |
| `Tab` | 自动补全命令和路径 |
| `↑/↓` | 切换历史命令 |

### 3.3 图片识别技巧

```
Ctrl + V 后粘贴截图
```

Claude 可以直接识别：
- 截图中的代码（即使是小图）
- 错误信息和堆栈跟踪
- UI 设计图
- 文档内容

**使用场景**:
- 快速分享错误信息
- 从截图中提取代码
- 分析 UI 设计

## 4. 使用建议与最佳实践

### 4.1 命令组合策略

| 场景 | 推荐组合 |
|-----|---------|
| **错误恢复** | `/rewind` 回退后继续 |
| **方案探索** | `/branch` 创建分支尝试不同方案 |
| **并行处理** | `/btw` 插入并行问题 (OMC) |
| **代码审查** | `/simplify` 快速审查 (OMC) |
| **长任务监控** | `/loop` 定时检查 (OMC) |

### 4.2 效率提升路线图

**第一阶段（掌握基础）**
- [ ] 熟练使用 `/rewind` 和 `/branch`
- [ ] 学会用 `Ctrl + V` 粘贴图片
- [ ] 掌握多行输入快捷键

**第二阶段（提升效率）**
- [ ] 使用 `/export` 保存重要对话
- [ ] 用 `/btw` 处理并行问题 (OMC)
- [ ] 用 `/simplify` 辅助代码审查 (OMC)

**第三阶段（高级用户）**
- [ ] 搭建 `/loop` 监控工作流 (OMC)
- [ ] 使用 `/schedule` 管理计划任务 (OMC)
- [ ] 结合 OMC 框架实现自动化

# 整体架构建议

> ⏱ 适用水平: 进阶 | 重要性: ★★★ | 推荐学习时间: 1-2天

---

## 1. 项目结构最佳实践

### 1.1 推荐目录结构

```
project/
├── .claude/                    # Claude Code 配置目录
│   ├── skills/                 # 自定义 Skills
│   │   ├── README.md
│   │   └── custom-skill.md
│   ├── hooks/                  # 自定义 Hooks
│   │   ├── index.js
│   │   ├── security.js
│   │   ├── audit.js
│   │   └── context.js
│   ├── memories/               # 记忆文件
│   │   ├── project-memory.json
│   │   └── context/
│   ├── plans/                  # 规划文件
│   └── .claudeignore           # 忽略文件
│
├── src/                        # 源代码
├── tests/                      # 测试文件
├── docs/                       # 文档
├── CLAUDE.md                   # 项目级指令
├── AGENTS.md                   # Agent 配置
├── .env                        # 环境变量
├── .gitignore
└── package.json
```

### 1.2 分层结构 (大型项目)

```
project/
├── apps/
│   ├── web/                    # Web 应用
│   ├── api/                    # API 服务
│   └── admin/                  # 管理后台
│
├── packages/                   # 共享包
│   ├── ui/                     # UI 组件库
│   ├── utils/                  # 工具函数
│   └── config/                 # 共享配置
│
├── .claude/                    # 统一配置
│   ├── skills/                 # 全局 Skills
│   ├── hooks/                  # 全局 Hooks
│   └── memories/               # 全局记忆
│
├── CLAUDE.md                   # 根目录项目说明
├── apps/
│   └── web/
│       └── CLAUDE.md          # Web 应用特定说明
└── packages/
    └── ui/
        └── CLAUDE.md          # 组件库特定说明
```

### 1.3 monorepo 结构

```
monorepo/
├── .claude/                    # 根级配置
│   ├── CLAUDE.md              # 全局项目说明
│   ├── skills/                # 跨项目 Skills
│   └── hooks/                 # 跨项目 Hooks
│
├── apps/
│   └── web/
│       └── .claude/           # 应用级配置
│           └── CLAUDE.md
│
├── packages/
│   ├── shared/
│   │   └── .claude/
│   │       └── CLAUDE.md
│   └── config/
│       └── .claude/
│           └── CLAUDE.md
│
└── scripts/                    # 构建脚本
```

## 2. CLAUDE.md 配置建议

### 2.1 CLAUDE.md 基础模板

```markdown
# 项目名称

## 项目概述
简要描述项目功能和技术栈

## 技术栈
- 前端: React 18 + TypeScript
- 后端: Node.js + Express
- 数据库: PostgreSQL
- 部署: Docker + Kubernetes

## 目录结构
```
src/           # 源代码
tests/         # 测试
docs/          # 文档
scripts/       # 脚本
```

## 开发规范

### 代码风格
- 使用 ESLint + Prettier
- TypeScript 严格模式
- 函数式组件 + Hooks

### Git 规范
- 提交信息格式: type(scope): description
- 分支命名: feature/xxx, bugfix/xxx
- PR 需要至少 1 个 review

### 测试规范
- 单元测试覆盖率 > 70%
- 集成测试覆盖核心流程
- E2E 测试覆盖关键路径

## 常用命令

```bash
npm install      # 安装依赖
npm run dev      # 开发模式
npm run build    # 构建
npm test         # 运行测试
npm run lint     # 代码检查
```

## 注意事项
- 避免直接修改 shared 包
- 生产环境配置在 .env.production
- 敏感信息不要提交到仓库
```

### 2.2 CLAUDE.md 高级配置

```markdown
# 项目名称

## 角色定义
你是一个 [职位]，专门负责 [职责]。你有以下特点：
- 熟悉 [技术栈]
- 注重 [质量标准]
- 擅长 [具体领域]

## 行为准则
1. 写代码前先分析，设计优先
2. 保证测试覆盖率
3. 遵循代码规范
4. 及时更新文档

## 约束条件
- 不允许直接提交到 main 分支
- 禁止删除生产数据
- 必须通过 CI 才能合并

## 上下文
- 项目阶段: [开发/测试/生产]
- 发布周期: 每两周一次
- 团队规模: [人数]

## 快捷方式
- /test: 运行测试
- /lint: 代码检查
- /build: 构建项目
- /deploy: 部署
```

### 2.3 CLAUDE.md 各层级配置

#### 项目根目录 CLAUDE.md
```markdown
# 项目级配置
全局项目说明、技术栈、团队规范
```

#### 应用目录 CLAUDE.md
```markdown
# 应用级配置
应用特定逻辑、业务规则、依赖关系
```

#### 模块目录 CLAUDE.md
```markdown
# 模块级配置
模块职责、接口定义、注意事项
```

## 3. AGENTS.md 配置

### 3.1 AGENTS.md 基础结构

```markdown
# Agent 配置文件

## 可用 Agent

| Agent | 模型 | 用途 |
|-------|------|------|
| architect | opus | 架构设计、复杂分析 |
| planner | sonnet | 任务规划、代码生成 |
| executor | sonnet | 代码实现、测试 |
| reviewer | opus | 代码审查、质量把控 |

## 默认 Agent 配置

```yaml
default:
  model: sonnet
  temperature: 0.7
  max_tokens: 4096

architect:
  model: opus
  temperature: 0.5
  max_tokens: 8192

reviewer:
  model: opus
  temperature: 0.3
  max_tokens: 4096
```

## Agent 调用规则

### 场景 1: 新功能开发
```
planner → executor → reviewer
```

### 场景 2: 架构重构
```
architect → planner → executor
```

### 场景 3: Bug 修复
```
executor (直接修复) → reviewer
```

## 协作拓扑

```yaml
team:
  size: 3
  agents:
    - architect
    - executor
    - reviewer
  communication: sequential
```
```

### 3.2 Agent 职责定义

```markdown
## Agent 职责矩阵

| 任务类型 | 主要 Agent | 辅助 Agent | 审核 Agent |
|---------|-----------|-----------|-----------|
| 新功能 | executor | architect | reviewer |
| 重构 | architect | executor | reviewer |
| Bug 修复 | executor | - | reviewer |
| 文档 | writer | - | reviewer |
| 部署 | executor | - | - |
```

## 4. 目录组织最佳实践

### 4.1 按功能分 vs 按类型分

#### 按功能分 (推荐大型项目)
```
src/
├── features/
│   ├── auth/
│   │   ├── components/
│   │   ├── hooks/
│   │   ├── services/
│   │   └── tests/
│   ├── dashboard/
│   └── settings/
└── shared/
    ├── components/
    └── utils/
```

#### 按类型分 (推荐小型项目)
```
src/
├── components/       # 组件
├── hooks/           # Hooks
├── services/        # 服务
├── utils/           # 工具
├── types/           # 类型定义
└── constants/       # 常量
```

### 4.2 .claudeignore 配置

```gitignore
# .claudeignore
node_modules/
dist/
build/
coverage/
*.log
.env
.env.local
.vscode/
.idea/
.git/
```

### 4.3 记忆文件组织

```json
// .claude/memories/project-memory.json
{
  "projectName": "项目名称",
  "techStack": ["React", "TypeScript"],
  "conventions": {
    "commitFormat": "type(scope): description",
    "branchNaming": "feature/xxx"
  },
  "team": {
    "members": ["Alice", "Bob"],
    "contacts": {}
  },
  "importantFiles": {
    "readme": "README.md",
    "docs": "docs/"
  }
}

## 6. 工作流集成

### 6.1 Git 工作流

```bash
# Feature 分支工作流
git checkout -b feature/xxx
# 开发...
git commit -m "feat(scope): description"
git push origin feature/xxx
# 创建 PR → Review → Merge
```

### 6.2 CI/CD 集成

```yaml
# .github/workflows/ci.yml
name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Run tests
        run: npm test
      - name: Run lint
        run: npm run lint
```

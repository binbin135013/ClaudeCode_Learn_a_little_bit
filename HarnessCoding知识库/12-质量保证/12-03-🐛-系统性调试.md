# 🐛 系统性调试

## 1. 概述

系统性调试是一套结构化的问题排查方法论，帮助快速定位和解决 Claude Code 运行时的各类问题。

## 2. 系统性调试套路 (superpowers:systematic-debugging)

### 2.1 调试工作流

```
┌─────────────────────────────────────────────────────────────┐
│                    系统性调试工作流                           │
│                                                             │
│  复现问题 ──▶ 收集信息 ──▶ 形成假设 ──▶ 验证假设            │
│      │            │              │             │           │
│      ▼            ▼              ▼             ▼           │
│  稳定复现    日志/环境      缩小范围      定位根因          │
│                                                             │
│  修复验证 ──▶ 总结归纳                                      │
│      │            │                                         │
│      ▼            ▼                                         │
│  确认修复    记录经验教训                                    │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 复现问题

**目标**：稳定复现问题，而非一次性偶发

| 复现策略 | 适用场景 | 操作 |
|---------|---------|------|
| 最小复现 | Bug明确时 | 提取核心代码单独运行 |
| 多次尝试 | 偶发问题时 | 多次触发确认复现率 |
| 环境隔离 | 环境相关时 | 在干净环境测试 |

**复现记录模板**：
```markdown
## 问题复现记录

**环境**:
- OS: [操作系统版本]
- Claude Code版本: [版本号]
- Node版本: [版本号]

**复现步骤**:
1. [步骤1]
2. [步骤2]
3. [步骤3]

**预期行为**: [应该发生什么]
**实际行为**: [实际发生了什么]
**复现率**: [100%/50%/偶发]
```

### 2.3 收集信息

**关键信息来源**

| 信息类型 | 获取方式 | 工具 |
|---------|---------|------|
| 错误日志 | 控制台输出 | Bash |
| 系统日志 | 日志文件 | Read |
| 环境状态 | 配置和变量 | Bash/env |
| 调用栈 | 异常堆栈 | 日志 |
| 性能数据 | 耗时监控 | profiler |

**常用诊断命令**：
```bash
# 查看Claude Code版本
claude --version

# 查看详细错误信息
claude --verbose

# 检查配置
claude config list

# 检查日志
cat ~/.claude/logs/*.log
```

### 2.4 形成假设

**假设来源**：
- 类似问题的历史解决方案
- 代码逻辑分析
- 错误模式识别

**假设记录**：
```markdown
## 假设清单

| # | 假设 | 可能性 | 验证方法 |
|---|-----|-------|---------|
| 1 | [假设内容] | 高/中/低 | [验证方式] |
| 2 | [假设内容] | 高/中/低 | [验证方式] |
```

### 2.5 验证假设

**验证原则**：
- 从高可能性假设开始
- 每次只改变一个变量
- 记录每次验证结果

## 3. 常见 Claude Code 运行时问题排查

### 3.1 启动问题

| 问题 | 可能原因 | 排查步骤 | 解决方案 |
|-----|---------|---------|---------|
| **CLI无响应** | 权限问题 | 检查 `--verbose` 输出 | 重新安装 |
| **版本不匹配** | 缓存问题 | `claude --version` vs 官方 | 清除缓存 |
| **端口被占用** | 进程冲突 | `netstat` 检查端口 | 杀进程或换端口 |
| **配置加载失败** | JSON格式错误 | 检查 `~/.claude/settings.json` | 修复JSON |

### 3.2 执行问题

| 问题 | 可能原因 | 排查步骤 | 解决方案 |
|-----|---------|---------|---------|
| **Tool调用失败** | 权限不足 | 检查Permission配置 | 提升权限 |
| **上下文溢出** | Token超限 | 检查Context大小 | 压缩或分段 |
| **异步操作卡住** | Promise未resolve | 添加超时 | 检查回调 |
| **状态不一致** | 并发问题 | 检查State管理 | 添加锁机制 |

### 3.3 输出问题

| 问题 | 可能原因 | 排查步骤 | 解决方案 |
|-----|---------|---------|---------|
| **无输出** | 流被阻断 | 检查输出流 | 重启会话 |
| **乱码** | 编码问题 | 检查Terminal编码 | 设置UTF-8 |
| **截断输出** | buffer限制 | 分段输出 | 增加buffer |
| **延迟高** | 网络/性能 | 监控耗时 | 优化或缓存 |

### 3.4 权限问题排查

```bash
# 1. 检查当前权限级别
claude config get permissionLevel

# 2. 检查允许的操作
claude permission list

# 3. 测试具体权限
claude --test-write

# 4. 重置权限（谨慎）
claude permission reset
```

## 4. 日志分析和调试工具

### 4.1 日志位置

| 平台 | 日志路径 |
|-----|---------|
| macOS | `~/.claude/logs/` |
| Linux | `~/.claude/logs/` |
| Windows | `%USERPROFILE%\.claude\logs\` |

### 4.2 日志级别

| 级别 | 含义 | 使用场景 |
|-----|------|---------|
| ERROR | 错误 | 异常情况 |
| WARN | 警告 | 潜在问题 |
| INFO | 信息 | 正常流程 |
| DEBUG | 调试 | 详细排查 |

### 4.3 日志分析技巧

**实时监控**：
```bash
# 实时查看日志
tail -f ~/.claude/logs/*.log

# 过滤错误
grep -i error ~/.claude/logs/*.log

# 查看最近日志
ls -lt ~/.claude/logs/ | head -5
```

**模式识别**：
```markdown
## 日志分析检查清单

- [ ] 时间戳：问题何时开始
- [ ] 错误码：特定错误模式
- [ ] 调用链：问题传播路径
- [ ] 上下文：用户操作序列
- [ ] 频率：偶发 vs 必现
```

### 4.4 调试工具

| 工具 | 用途 | 使用方式 |
|-----|------|---------|
| `--verbose` | 详细输出 | `claude --verbose [command]` |
| `--dry-run` | 不执行 | `claude --dry-run [command]` |
| `DEBUG=*` | 调试模式 | `DEBUG=* claude [command]` |
| Inspector | 可视化调试 | 内置工具 |

## 5. 回归保护方法

### 5.1 回归测试策略

```
┌─────────────────────────────────────────────────────────────┐
│                    回归保护层级                               │
│                                                             │
│  单元测试 (快) ──▶ 集成测试 (中) ──▶ E2E测试 (慢)          │
│      │              │               │                      │
│      ▼              ▼               ▼                      │
│   函数级别       模块级别         完整流程                  │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 回归测试优先级

| 优先级 | 测试类型 | 覆盖范围 | 运行频率 |
|-------|---------|---------|---------|
| P0 | 核心功能测试 | 阻断Bug修复 | 每次提交 |
| P1 | 高风险功能 | 主要业务流程 | 每日构建 |
| P2 | 中风险功能 | 次要功能 | 每周构建 |
| P3 | 低风险功能 | 边缘场景 | 发布前 |

### 5.3 自动化回归保护

```yaml
# CI配置示例
regression:
  on_push:
    - run: npm run test:unit
    - run: npm run test:integration
    - run: npm run test:e2e

  thresholds:
    coverage: 80
    max_duration: 300  # seconds
```

### 5.4 回归保护检查清单

**代码变更前**：
- [ ] 了解变更影响范围
- [ ] 确定需要执行的测试
- [ ] 备份当前状态

**代码变更后**：
- [ ] 执行核心测试套件
- [ ] 验证问题确实修复
- [ ] 确认没有引入新问题

**发布前**：
- [ ] 执行完整测试套件
- [ ] 手动验证关键路径
- [ ] 准备回滚方案

## 6. 调试模式与技巧

### 6.1 二分查找定位法

**适用场景**：不确定问题出现在哪个环节

```
怀疑范围: A → B → C → D → E

1. 从中间测试: C 是否正常？
   ├─ 是 → 问题在后半段 D/E
   └─ 否 → 问题在前半段 A/B/C
2. 继续二分直到定位
```

### 6.2 消减法

**适用场景**：问题由多个因素共同导致

```
原始状态: 因素1 + 因素2 + 因素3 + 因素4 = 问题
         ↓ 移除因素4
因素1 + 因素2 + 因素3 = 问题仍存在
         ↓ 移除因素3
因素1 + 因素2 = 问题消失
         ↓
结论: 因素3 是问题根源
```

### 6.3 对比法

**适用场景**：之前正常现在出问题

```bash
# 对比正常和异常状态
git diff HEAD~1  # 查看最近变更

# 对比配置
diff ~/.claude/config.normal ~/.claude/config.current

# 对比环境
env | sort > /tmp/env.current
```

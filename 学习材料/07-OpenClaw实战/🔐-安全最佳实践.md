# 🔐 OpenClaw 安全最佳实践

> ⏱ 难度: ★★☆ | 重要性: ★★★ | 推荐学习时间: 1-2天
> ⚠️ **特别重要**: 如果你要公网部署 OpenClaw，这篇必看！

---

## 概述

### 安全层级

```
网络层 → TLS 1.3 + HTTPS
认证层 → Gateway Token + DM 配对
权限层 → allowFrom + 工具权限
执行层 → Sandbox 沙箱隔离
审计层 → 日志 + Webhook 通知
```

---

## 核心安全配置

### 1. Gateway Token

```json5
// openclaw.json
{
  "gateway": {
    "token": "your-secret-token-here"  // WebSocket API 认证
  }
}
```

### 2. DM 配对策略

| 策略 | 说明 | 安全级别 |
|------|------|---------|
| `pairing` | 新用户需验证码 | ✅ 推荐 |
| `allowFrom` | 白名单指定用户 | ✅ 最安全 |
| `open` | 所有人可用 | ❌ 危险 |

### 3. TLS 配置

```nginx
server {
    listen 443 ssl http2;
    ssl_protocols TLSv1.3;
    ssl_certificate /etc/nginx/certs/cert.pem;
    ssl_certificate_key /etc/nginx/certs/key.pem;
}
```

### 4. 沙箱隔离

限制 AI 可执行的操作范围：
- 禁止 `rm -rf`
- 禁止网络访问内网
- 限制文件操作目录

---

## 安全检查清单

> [!checklist] 部署前必查
> - [ ] Gateway Token 已设置
> - [ ] TLS 1.3 已启用
> - [ ] DM 策略设为 pairing 或 allowFrom
> - [ ] 沙箱隔离已启用
> - [ ] 环境变量不包含明文密钥
> - [ ] 日志不记录敏感信息
> - [ ] 定期更新 OpenClaw 版本

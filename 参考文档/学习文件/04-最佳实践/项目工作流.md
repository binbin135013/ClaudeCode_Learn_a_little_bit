# 项目工作流

> [!summary] 一句话定义
> 使用 Claude Code 的**标准化项目结构和开发流程**，让 AI 助手和团队成员高效协作。

---

## 推荐项目结构

```
project-root/
├── .claude/                  # Claude Code 配置
│   ├── settings.json         # 权限和工具配置
│   ├── settings.local.json   # 本地覆盖（不入库）
│   ├── commands/             # 自定义 /slash 命令
│   ├── hooks/                # 生命周期钩子
│   └── skills/               # 技能包
├── .github/workflows/        # CI/CD（[[CI-CD与自动化]]）
├── docs/ai-context/          # AI 上下文文档
│   ├── project-structure.md
│   ├── coding-standards.md
│   └── architecture.md
├── src/                      # 源代码
├── tests/                    # 测试代码
├── CLAUDE.md                 # 主配置 → [[CLAUDE-md编写指南]]
├── .mcp.json                 # MCP 服务器 → [[MCP协议]]
└── .gitignore                # Git 忽略
```

## 配置入库策略

### 必须入库（团队共享）
```
!.claude/settings.json
!.claude/commands/
!.claude/hooks/
!.claude/skills/
!CLAUDE.md
!.mcp.json
```

### 禁止入库（个人本地）
```
.claude/settings.local.json
.env / .env.local
```

## 开发工作流

### 日常开发循环

```
1. 启动 Claude Code
   claude

2. 描述需求（自然语言）
   "给用户模块添加邮箱验证功能"

3. Claude 执行（需确认）
   ├── 分析现有代码
   ├── 生成修改方案
   ├── 等待确认后执行
   └── 运行测试验证

4. 审查和提交
   /review → 审查变更
   git commit → 提交代码
```

### 任务分解策略

> [!tip] 大任务拆分原则
> ```markdown
> # ❌ 不好：一次性大任务
> "重构整个项目的所有模块..."
>
> # ✅ 好：分步骤小任务
> 1. "先分析用户模块的当前结构"
> 2. "基于分析结果，提出重构方案"
> 3. "执行用户模块重构"
> 4. "验证重构结果"
> 5. "继续处理下一个模块..."
> ```

# 环境搭建指南

> [!summary] 一句话定义
> 从零开始搭建 Claude Code + OpenClaw 开发环境的**完整步骤**，包括前置条件、分步安装、集成配置和环境验证。

> [!note] 内容来源
> Claude Code 安装配置部分基于通用实践和官方文档综合，非直接引自项目源教程。OpenClaw 安装部分基于源教程。建议对照 [Claude Code 官方文档](https://docs.anthropic.com/en/docs/claude-code) 和 [OpenClaw 官方文档](https://github.com/openclaw/openclaw) 验证。

---

## 前置条件

| 依赖 | 最低版本 | 推荐方式 | 检查命令 |
|------|---------|---------|---------|
| Node.js | 22+ | nvm 安装 | `node --version` |
| npm | 10+ | 随 Node.js | `npm --version` |
| Git | 2.30+ | 官网下载 | `git --version` |
| API Key | - | Anthropic Console | - |
| Docker Desktop | 最新 | 可选，用于 OC 部署 | `docker --version` |

> [!tip] 为什么用 nvm
> nvm 允许你在多个 Node.js 版本之间切换，避免全局安装时的权限问题，也方便按项目使用不同版本。

## 一、Claude Code 环境搭建

### 步骤 1：安装 Claude Code

```bash
# 全局安装（推荐）
npm install -g @anthropic-ai/claude-code

# 验证安装
claude --version
```

> [!warning] Windows 用户注意
> 如果 PowerShell 报执行策略错误，先运行：
> `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser`

### 步骤 2：配置 API Key

```bash
# 方式 A：环境变量（推荐，安全）
export ANTHROPIC_API_KEY=sk-ant-xxxxx

# 方式 B：写入 shell 配置，永久生效
echo 'export ANTHROPIC_API_KEY=sk-ant-xxxxx' >> ~/.bashrc
source ~/.bashrc

# Windows PowerShell
[Environment]::SetEnvironmentVariable("ANTHROPIC_API_KEY", "sk-ant-xxxxx", "User")
```

> [!tip] 使用中转站
> 如果需要通过代理访问，额外配置：
> ```bash
> export ANTHROPIC_BASE_URL=https://your-proxy.example.com
> ```

### 步骤 3：创建全局 CLAUDE.md

```bash
# 创建全局配置目录
mkdir -p ~/.claude

# 创建全局 CLAUDE.md
cat > ~/.claude/CLAUDE.md << 'EOF'
# 个人偏好
- 使用中文交流
- 代码注释用中文
- 优先使用 TypeScript

# 常用技术栈
- 前端：React + Next.js
- 后端：Node.js + Express
- 数据库：PostgreSQL + Prisma
EOF
```

- 编写规范 → [[CLAUDE-md编写指南]]

### 步骤 4：安装推荐 MCP Servers

> [!tip] 按需安装
> 不要贪多，只装项目需要的。以下 3-5 个是大多数项目的起点。

```bash
# 文件系统 MCP（CC 内置，无需额外安装）
# GitHub MCP
npx @anthropic-ai/mcp-github

# 或在项目中创建 .mcp.json
cat > .mcp.json << 'EOF'
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-filesystem", "/path/to/project"]
    }
  }
}
EOF
```

常用 MCP Server 速查：

| Server | 用途 | 安装 |
|--------|------|------|
| filesystem | 文件读写 | `npx @anthropic-ai/mcp-filesystem` |
| github | PR/Issue 管理 | `npx @anthropic-ai/mcp-github` |
| postgres | 数据库查询 | `npx @anthropic-ai/mcp-postgres` |

- MCP 详细配置 → [[MCP协议]]

### 步骤 5：测试 Claude Code

```bash
# 创建测试项目
mkdir ~/test-cc && cd ~/test-cc
git init

# 启动 Claude Code
claude

# 输入测试指令
> 你好，请帮我创建一个 hello.txt 文件，内容是 "Hello Claude Code"

# 确认文件已创建
ls hello.txt
```

## 二、OpenClaw 环境搭建

### 步骤 1：安装 OpenClaw

```bash
# 全局安装
npm install -g openclaw

# 验证安装
openclaw --version
```

### 步骤 2：运行 onboard 向导

```bash
# 交互式初始化配置
openclaw onboard
```

向导会引导你完成：
- 选择 AI 模型提供商
- 输入 API Key
- 配置基本信息（语言、时区等）

> [!warning] 常见问题
> 如果 onboard 失败，运行 `openclaw doctor` 查看诊断信息。
> 大多数失败原因是 Node.js 版本过低（需要 22+）。

### 步骤 3：配置 openclaw.json

```bash
# 编辑主配置文件
# 路径：~/.openclaw/openclaw.json
```

核心配置项：

```json
{
  "name": "my-assistant",
  "language": "zh-CN",
  "timezone": "Asia/Shanghai",

  "agents": {
    "defaults": {
      "model": {
        "primary": "claude-sonnet-4-6",
        "fallbacks": ["claude-haiku-4-5"]
      }
    }
  },
  "apiKey": "${ANTHROPIC_API_KEY}",

  "gateway": {
    "port": 18789,
    "host": "127.0.0.1"
  },

  "channels": {
    "cli": { "enabled": true }
  }
}
```

- 模型配置详解 → [[模型配置策略]]

### 步骤 4：测试 OpenClaw

```bash
# CLI 模式测试
openclaw agent --message "你好，请自我介绍"

# 预期输出：AI 回复自我介绍
```

### 步骤 5：连接消息平台

以 Telegram 为例：

```bash
# 1. 在 Telegram 中创建 Bot（找 @BotFather）
# 2. 获取 Bot Token
# 3. 添加 Telegram Channel 配置
```

在 `openclaw.json` 中添加：

```json
{
  "channels": {
    "cli": { "enabled": true },
    "telegram": {
      "enabled": true,
      "token": "${TELEGRAM_BOT_TOKEN}"
    }
  }
}
```

```bash
# 启动 Gateway
openclaw gateway

# 在 Telegram 中给你的 Bot 发消息测试
```

- 其他平台配置 → [[安全最佳实践]]
- Docker 部署方式 → [[Docker部署]]

## 三、CC + OC 集成搭建

### 步骤 1：共享 API Key

让 Claude Code 和 OpenClaw 使用同一个 API Key，统一管理和计费。

```bash
# 确保环境变量已设置（Claude Code 步骤 2 中已配置）
echo $ANTHROPIC_API_KEY

# OpenClaw 的 openclaw.json 中已通过 ${ANTHROPIC_API_KEY} 引用
# 两者自动共享同一个 Key
```

> [!tip] 代理统一管理
> 如果两者都需要走代理，设置同一个 BASE_URL：
> ```bash
> export ANTHROPIC_BASE_URL=https://your-proxy.example.com
> export OPENCLAW_API_BASE_URL=https://your-proxy.example.com
> ```

### 步骤 2：配置 MCP 桥接

通过 MCP 让 Claude Code 能够调用 OpenClaw 的能力。

```bash
# 安装 mcporter（MCP 桥接工具）
npm install -g mcporter
```

在项目的 `.mcp.json` 中添加 OpenClaw 桥接：

```json
{
  "mcpServers": {
    "openclaw": {
      "command": "mcporter",
      "args": ["--gateway", "http://localhost:18789"]
    }
  }
}
```

> [!warning] 前提条件
> OpenClaw Gateway 必须处于运行状态（`openclaw gateway`）才能桥接成功。

### 步骤 3：测试双向通信

```bash
# 终端 1：启动 OpenClaw Gateway
openclaw gateway

# 终端 2：启动 Claude Code
cd ~/test-cc
claude

# 在 Claude Code 中测试
> 通过 OpenClaw MCP 发送消息 "从 CC 发来的测试消息"
```

## 环境验证清单

完成所有搭建后，逐项验证：

### Claude Code

- [ ] `claude --version` 输出正确版本号
- [ ] `echo $ANTHROPIC_API_KEY` 输出你的 API Key
- [ ] `~/.claude/CLAUDE.md` 文件存在且内容正确
- [ ] 新建项目后 `claude` 能正常启动对话
- [ ] AI 能正确读取和写入文件

### OpenClaw

- [ ] `openclaw --version` 输出正确版本号
- [ ] `openclaw doctor` 全部检查通过
- [ ] `~/.openclaw/openclaw.json` 配置正确
- [ ] `openclaw agent --message "测试"` 收到回复
- [ ] 消息平台（如 Telegram）能正常收发消息

### 集成

- [ ] Claude Code 和 OpenClaw 共享同一个 API Key
- [ ] `mcporter` 已安装
- [ ] Claude Code 中能通过 MCP 调用 OpenClaw
- [ ] OpenClaw Gateway 正常运行，端口 18789 可访问

### 安全

- [ ] API Key 未硬编码在任何配置文件中（使用环境变量）
- [ ] `~/.claude/settings.json` 配置了 `allowedTools`
- [ ] OpenClaw 配置了 DM 策略（非 `open`）
- [ ] `.env` 文件已加入 `.gitignore`
- [ ] Docker 部署时配置了资源限制

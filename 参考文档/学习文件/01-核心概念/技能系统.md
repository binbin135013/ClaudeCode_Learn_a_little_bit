# 技能系统 (Skills)

> [!summary] 一句话定义
> Claude Code 的**可复用功能包**，通过 `SKILL.md` 定义，类似手机 APP，一次创建、到处使用。

---

## 双生态技能体系

### Claude Code Skills

| 层级 | 位置 | 说明 |
|------|------|------|
| 全局 | `~/.claude/skills/` | 所有项目可用 |
| 项目 | `.claude/skills/` | 项目特定技能 |
| Community | oh-my-claudecode 等 | 社区技能包 |

**SKILL.md 结构**：
```markdown
---
name: my-skill
description: 技能描述
context: fork        # fork=独立上下文 | shared=共享上下文
---

# 技能指令内容
...
```

**关键变量**：`${CLAUDE_SKILL_DIR}` — 指向技能所在目录

### OpenClaw Skills

| 层级 | 来源 | 说明 |
|------|------|------|
| Bundled Skills | 核心仓库 | 随 OpenClaw 安装，50+ 内置技能 |
| Managed Skills | ClawHub (clawhub.com) | 社区技能市场 |
| Workspace Skills | 本地工作空间 | 自己写的技能 |

**热门内置技能**：`gog`(Google Workspace), `obsidian`, `coding-agent`, `github`, `spotify-player`

## 技能 vs 工具 vs 命令

> [!note] 三者区别
> - **工具 (Tools)** = 单个原子操作（读文件、发请求）
> - **技能 (Skills)** = 组合工具完成复杂任务的知识
> - **命令 (Commands)** = `/slash` 快捷触发方式

```
工具 (Tools)        → 读文件、搜索、HTTP请求
技能 (Skills)       → GitHub Issue 管理 = 搜索+读取+评论+标签
命令 (Commands)     → /review 触发代码审查技能
```

## 创建自定义技能

> [!tip] Claude Code 技能开发流程
> 1. 创建 `.claude/skills/my-skill/SKILL.md`
> 2. 编写技能指令（Markdown）
> 3. 测试：修改后 Hot Reload 自动加载
> 4. 分享：发布到社区或团队共享

详细开发指南 → [[Skills开发]] · [[自定义技能开发]]

## 进阶：双生态技能对照表

> [!important] 关键区别
> 虽然 Claude Code 和 OpenClaw 都有"技能"概念，但实现和使用方式有本质差异：

| 维度 | Claude Code Skills | OpenClaw Skills |
|------|-------------------|-----------------|
| **格式** | SKILL.md (Markdown) | Markdown 指令文件 |
| **上下文** | fork 或 shared 模式 | 在 Agent 推理循环中加载 |
| **触发** | 自动匹配或手动调用 | 工具调用链自动触发 |
| **组合** | 可嵌套调用 | 50+ 内置可组合 |
| **分发** | 团队 Git 共享 | ClawHub 市场 |
| **热更新** | Hot Reload | 需要 Gateway 重载 |

### 技能开发决策树

```
你的需求是什么？
├── 编程自动化 → Claude Code Skill
│   ├── 代码模板 → SKILL.md + context: fork
│   ├── 审查流程 → SKILL.md + context: shared
│   └── 工作流编排 → 结合 [[Hooks系统]]
├── 日常自动化 → OpenClaw Skill
│   ├── 信息聚合 → 内置 skill (weather, summarize)
│   ├── 平台操作 → 内置 skill (github, notion)
│   └── 自定义逻辑 → Workspace Skill
└── 跨平台 → [[MCP协议]] + 两者结合
```
